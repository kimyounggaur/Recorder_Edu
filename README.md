# 🎵 리코더 교실 — 보고, 만지고, 들으면서 배우는 운지법

소프라노 리코더 운지법 인터랙티브 학습 웹앱.
`MotionLab_UIUX_업그레이드_Fable5_프롬프트.md`의 Phase 0~8 설계 방법론을
`01 Source` 폴더의 교재 자료(자세·잡는 법·입 모양·구멍 막기·텅잉 + 운지 차트 7세트)에 적용해 개발했습니다.

## 실행 방법

빌드 도구가 필요 없습니다.

- **그냥 열기:** `index.html`을 더블클릭 (file:// 로도 동작)
- **로컬 서버:** `python -m http.server 8137` → http://localhost:8137
- **배포:** 폴더째 Vercel/Netlify/GitHub Pages에 올리면 끝 (정적 사이트)

파일 구성: `index.html` · `styles.css` · `data.js`(운지 데이터) · `app.js`(코어) · `views.js`(화면)
— 외부 라이브러리 0개, 웹폰트(Jua) 1개(미로딩 시 시스템 폰트로 대체).

## Phase 1 — 정보 구조

"무엇을 하고 싶은가" 기준 8개 섹션:
**홈(놀이터 히어로) → 준비하기(5) → 운지 배우기(7세트) → 운지표(26음 갤러리) → 운지 놀이터 → 연습하기(퀴즈 2종+연주 4곡) → 보관함 → 가이드**

모든 음에 태그: 옥타브(낮은음/높은음/사이음) · 손(왼손/양손) · 난이도(입문/기초/도전) · 운지 체계(독일식 G/바로크식 B).

## 주요 기능 (Phase 2~6)

- **운지 놀이터**: 구멍을 누르면 운지를 인식해 음 이름 표시 + 실제 음높이 재생(Web Audio 합성).
  틀리면 "혹시 ○를 만들고 있나요? n번만 바꾸면 돼요" 근접 힌트. 뒤·6·7번은 막기→반만→열기 3단계.
- **검색/필터**: 운지표 실시간 검색(디바운스), 멀티 필터 칩, 갤러리↔목록 토글, 행동 유도형 빈 상태.
- **커맨드 팔레트** `Ctrl+K`: 음/세트/노래/화면 퍼지 검색, 최근·즐겨찾기 노출, 완전한 키보드 조작.
- **온보딩**: 첫 방문 3스텝 코치마크(건너뛰기 가능, localStorage 기억, 가이드에서 다시 보기),
  추천 시작점 4단계, '이어서 하기' 재방문 띠.
- **표준 음 카드**: 악보(8va 보표) + 리코더 운지 + 태그 + 들어보기/자세히. 상세 모달에 색상 운지 설명,
  "왜 이렇게 불까요?" 팁, 이명동음 안내, G/B 전환.
- **저장(전부 localStorage, 로그인 없음)**: 즐겨찾기 ★ / 최근 본 항목 / 나만의 모음 / 진행률(준비·세트·들어본 음·완주 곡) /
  퀴즈 최고 기록 / JSON 내보내기·가져오기·전체 초기화. 저장 실패(쿼터) 예외 처리 + 메모리 폴백.
- **연습**: 운지→음 / 음→운지 퀴즈(범위 5종, 10문제, 정답 사운드, 최고 기록),
  따라 연주 4곡(비행기·환희의 송가는 왼손만!) — 음 리본+가사, 한 음씩 모드, 속도 60~140%.
- **모바일**: 하단 탭 바 + 드로어, 44px 터치 히트 영역(보이지 않는 히트 서클), 1~3열 유동 그리드,
  좁은 화면 헤더 다이어트.

## 운지 데이터 출처와 검증

- 26개 음(C5~D7) 운지는 `01 Source`의 차트 스크린샷 7장을 1차 기준으로 디지타이즈.
- 독일식/바로크식이 다른 음: **파(F5) · 파♯(F♯5) · 높은 파(F6) · 높은 파♯(F♯6) · 높은 솔♯(G♯6)**.
- 화면 판독이 모호했던 일부(높은 파 바로크식 6번)는 표준 공인 차트(Ø·1·2·3·4·6)를 따름.
- 음높이는 평균율 A4=440Hz 기준.
